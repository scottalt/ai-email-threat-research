# v2.0 Production Deployment Checklist

## Pre-deployment checks

### 1. Check for duplicate callsigns in prod
```sql
SELECT display_name, COUNT(*) FROM players GROUP BY display_name HAVING COUNT(*) > 1;
```
If duplicates exist, resolve them before applying the unique constraint migration.

### 2. Merge branches
```
v2.0-bugfix → v2.0 → master
```

---

## Database migrations (run in prod Supabase SQL Editor, in order)

Each migration is in `supabase/migrations/`. Run them sequentially:

| # | Migration file | What it does |
|---|---------------|-------------|
| 1 | `20260322200000_create-h2h-tables.sql` | H2H matches, match answers, player stats tables |
| 2 | `20260322220000_add-theme-to-players.sql` | `theme_id` column on players |
| 3 | `20260322230000_add-profile-social-fields.sql` | `featured_badge`, `featured_badges`, `bio`, `privacy_level` on players |
| 4 | `20260322240000_unique-callsign-constraint.sql` | Unique constraint on `display_name` (requires no dupes — see step 1) |
| 5 | `20260323300000_create-content-registry.sql` | 9 `registry_*` tables (completely isolated) |

**Risk:** All migrations add new tables or columns with defaults. Zero impact on existing data. The only risky one is #4 (unique constraint) — verify no dupes first.

---

## Seed content registry on prod

After migration #5, seed the registry:

```bash
SUPABASE_URL=<prod-url> SUPABASE_SERVICE_ROLE_KEY=<prod-service-role-key> npx tsx scripts/seed-content-registry.ts
```

Or source from Vercel:
```bash
npx vercel env pull .env.prod --environment=production
# Then manually run with prod vars
```

## Push to production

```bash
git checkout v2.0
git merge v2.0-bugfix
git checkout master
git merge v2.0
# Bump version in package.json to 2.0.0
# Add changelog entry
git tag v2.0.0
git push origin master --tags
```

Vercel auto-deploys master to prod.

## Post-deployment verification

1. **Auth flow** — sign in on prod, verify terms checkbox works for new users
2. **PvP** — queue for a match, verify bot match triggers after 30s
3. **Profile** — check all tabs (INFO, SOLO STATS, PvP STATS, QUESTS, FRIENDS)
4. **Admin** — visit `/admin/content`, verify all 8 tabs load with seeded data
5. **Privacy/Terms** — visit `/privacy` and `/terms`, check footer links
6. **Inventory** — themes and badges display correctly
7. **Leaderboard** — H2H leaderboard loads

## Rollback plan

If something breaks:
```bash
# Revert the merge commit on master
git revert HEAD
git push origin master
```

DB migrations are additive (new tables/columns) — they don't need rollback. The old code simply ignores the new columns.
