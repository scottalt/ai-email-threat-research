TECHNIQUE: hyper-personalization

Definition: Uses specific personal context — recipient's name, company, role, team, current project, or recent events — to build false credibility. This is a primary GenAI differentiator.

Difficulty calibration:
- Easy: Uses recipient name only. Generic role references ("as a member of the IT team"). Personalization is surface-level and the phishing intent is still obvious.
- Medium: References name, company, and role naturally. May reference department or job function. Feels more personal than a template.
- Hard: References specific-sounding project names, internal systems, team members, or recent events. Creates the impression that only someone with real insider access could have sent this.
- Extreme: Reads like a genuine follow-up to a real conversation. References specific dates, meeting topics, action items. The personalization is so deep it creates strong social proof. The only tells are forensic (sender domain, unusual request).

Anti-pattern rules:
- Do NOT just add a name and call it personalized — real personalization references context the recipient would recognize
- For hard/extreme: create fictional but realistic project names, system names, and team contexts
- The phishing intent should be HIDDEN BEHIND the personalization — the personal context is the camouflage
- Vary contexts: colleagues, clients, vendors, recruiters, professional networks
